◆ LEUART0_IRQHandler()

void LEUART0_IRQHandler ( void  )

IRQ handler for LEUART0.

This is an IRQ handler for LEUART0. It is used to determine when the LEUART0 is available to transmit and when the LEUART0 has completed transmission.

It uses the TXBL and TXC interrupts.

Note
The BLE test uses polling, not interrupts to function. Normal BLE functionality will use interrupts in order to maintain low energy operation.

uint8_t leuart_app_receive_byte ( LEUART_TypeDef *  leuart)

LEUART App Receive Byte polls a receive byte for the LEUART TDD test.

The BLE module will respond to AT commands if the BLE module is not connected to the phone app. To validate the minimal functionality of the LEUART peripheral, write and reads to the LEUART will be performed by polling and not interrupts.

Note
In polling a receive byte, a while statement checking for the RXDATAV bit in the Interrupt Flag register is required before reading the RXDATA register.
Parameters
[in]leuartDefines the LEUART peripheral to access.
Returns
Returns the byte read from the LEUART peripheral

LEUART driver.

This module contains all the functions to support the driver's state machine to transmit a string of data across the LEUART bus. There are additional functions to support the Test Driven Development test that is used to validate the basic set up of the LEUART peripheral. The TDD test for this class assumes that the LEUART is connected to the HM-18 BLE module. These TDD support functions could be used for any TDD test to validate the correct setup of the LEUART.

Function to open Low Energy UART peripheral communication port.

This routine initializes a LEUART port, routes the RX and TX pins, and enables / clears RX and TX and their respective buffers as needed.

Parameters
[in]leuartPointer to the base peripheral address of the LEUART peripheral being used.
[in]leuart_settingsPointer to the LEUART_OPEN_STRUCT containing the desired settings for this instance of the LEUART port.

◆ leuart_start()

void leuart_start ( LEUART_TypeDef *  leuart,
char *  string,
uint32_t  string_len 
)

Function to start Low Energy UART peripheral communication port write operation.

This routine initializes an leuart_payload which stores the state of the LEUART transmission. All information required by the LEUART state machine interacts with this LEUART payload struct.

Once the payload is initialized, this function enables the TXBL interrupt, which initiates the transmit sequence.

Note
This function must only be called when the state of the transmit state machine is in IDLE mode and when the LEUART peripheral is also in an IDLE state.
Parameters
[in]leuartPointer to the base peripheral address of the LEUART peripheral being used.
[in]stringThe string which must be transmitted over UART.
[in]string_lenThe number of characters in the string to be transmitted.
